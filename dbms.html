Subject - Data Base Management System

Course Overview

Database Management System (DBMS) is a software suite that provides an interface for managing and interacting with databases. Its primary purpose is to facilitate the efficient storage, retrieval, and management of data in a structured way. Here's an overview of key concepts related to DBMS:

  • Data Base: A structured collection of data organized to meet specific requirements. Databases can be relational, hierarchical, network, or object-oriented, among other types.
  • DBMS Components:
  • Data Definition Language (DDL):
    • Allows users to define the structure and schema of the database. Includes commands like CREATE, ALTER, and DROP for defining tables and relationships.
  • Data Manipulation Language (DML):
  • Facilitates the manipulation of data stored in the database. Includes commands like SELECT, INSERT, UPDATE, and DELETE for querying and modifying data.
  • Database Engine:
  • Core component responsible for managing storage, retrieval, and processing of data. Optimizes queries and transactions for performance.
  • Query Processor:
  • Analyzes and processes SQL queries to generate execution plans. Optimizes query performance by choosing efficient algorithms.
  • Database Security:
  • Enforces access controls, authentication, and authorization to secure data. Protects against unauthorized access and data breaches.
  • Backup and Recovery:
  • Manages the backup and restoration of data to prevent data loss. Implements recovery mechanisms to restore the database to a consistent state after failures.
  • Relational Databases: Most common type of database where data is organized in tables with rows and columns. Relationships between tables are established using keys.
  • Normalization: Process of organizing data to eliminate redundancy and improve data integrity. Divides large tables into smaller, related tables.
  • Structured Query Language (SQL): Standardized language used to interact with relational databases. Allows users to define, manipulate, and query data.
  • NoSQL Databases: Designed to handle unstructured or semi-structured data and provide flexibility in data modeling. Types include document-oriented, key-value, column-family, and graph databases.

Unit 1: Structured Query Language (SQL)

Structured Query Language (SQL) is a powerful and standardized programming language specifically designed for managing and manipulating relational databases. SQL allows users to define, query, update, and manage databases, providing a comprehensive set of commands for creating tables, inserting data, querying information, and ensuring data integrity through constraints and transactions. It serves as the fundamental language for interacting with relational database management systems (RDBMS) like MySQL, PostgreSQL, Oracle, and Microsoft SQL Server.

Unit 2: ER Diagram And Relational Algebra

Entity-Relationship (ER) diagrams serve as visual models for designing relational databases, portraying entities, their attributes, and the relationships between them. Entities are depicted as rectangles, attributes as ovals, and relationships as diamond shapes connecting entities. This visualization aids in structuring databases, ensuring data integrity, and optimizing relational designs. In parallel, relational algebra provides a theoretical framework for manipulating relational databases through operations such as selection (σ), projection (π), union (∪), intersection (∩), join (⨝), and difference (-). These operations are crucial for querying and transforming data within relational databases. Relational algebra forms the foundation for query languages like SQL, enabling effective expression and optimization of queries on relational databases.

Unit 3: Normalization

Normalization is the database design process to minimize data redundancy and dependency by organizing tables and attributes. It involves decomposing complex tables into smaller, well-structured ones, reducing anomalies and ensuring efficient data storage while maintaining data integrity.

Unit 4: NoSQL

NoSQL, or "Not Only SQL," is a category of database systems that diverge from traditional relational databases by not using a structured query language (SQL) for data manipulation. Instead, NoSQL databases are designed to handle diverse and unstructured data types, providing flexibility and scalability for applications with rapidly evolving data requirements. They can use various data models, such as document-oriented, key-value, column-family, or graph databases, catering to specific use cases and offering high performance and horizontal scalability.

Unit 5: Transactions

Transactions in databases ensure that a series of operations are treated as a single, indivisible unit, following the ACID properties for Atomicity, Consistency, Isolation, and Durability. They provide reliability and maintain data integrity by guaranteeing either the successful completion or full rollback of a set of related operations.
